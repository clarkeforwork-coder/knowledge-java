# 筆記模板

兩軌各一套模板：🔰 走教學式結構，🔬 走問題驅動結構。

## 🔰 基礎筆記

```markdown
# （主題名稱）

> **情境**：從一個會實際踩到的問題或錯誤輸出切入，2～4 行。
> 可以是一段出錯的程式碼、一個直覺以為對但其實錯的寫法、
> 或一個工作上會遇到的需求。讓讀者先「有感」再進入概念。

## （概念段落 × N）

每段原則：

- **先程式碼、後解釋** —— 範例在前，文字說明在後
- 對比類主題用表格（如 Checked vs Runtime Exception）
- 反模式用 ❌/✅ 標注在程式碼註解裡
- snippet 力求「複製進 main 就能跑」

## 常見陷阱

有才寫，不硬湊。

## 重點

- 3～5 條帶走訊息，一條一行

## 常見面試題

1～3 題經典問法，附一句話方向提示（不寫完整解答，答案就在上文）。

## 延伸閱讀

- 官方文件、JEP、經典文章優先；避免易失效的部落格連結

---

🔬 想深入：[（深入文標題）](deep-xxx.md)　← 有對應深入文才放
```

## 🔬 深入筆記（檔名 `deep-` 開頭）

```markdown
# （主題名稱）

## 問題起點

為什麼會好奇這個？從 🔰 筆記留下的懸念、一個反直覺的行為、
或一個效能數字出發。

## 追蹤過程

原始碼追讀／實驗／benchmark。這是深入文的主體，
要呈現「怎麼查到的」，不只是結論。

## 結論

回答問題起點提出的疑問。

## 回到實務

這個底層知識改變了什麼日常決策？
（例如：知道 HashMap 擴容成本後，該怎麼給初始容量）

## 延伸閱讀
```

## 可執行範例（混合制）

- 預設文內 snippet 即可，**不**另附範例檔。
- 只有「不跑看不出結果」的主題才附：並發競態、效能對比、GC 行為等。
- 附檔放在該章的 `examples/` 子目錄，例如 `07-concurrency/examples/RaceConditionDemo.java`。
- 每個範例是**單一 .java 檔**，用 `java RaceConditionDemo.java`（JDK 11+ 單檔執行）就能跑，不引入建置工具。
- 筆記內文用一行連過去：「▶ 可執行範例：[RaceConditionDemo.java](examples/RaceConditionDemo.java)」。
