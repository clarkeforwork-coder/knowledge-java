# knowledge-java 專案規則

Java 技術筆記 repo（教育訓練教材的通用化公開版，也是 portfolio 的一部分）。
筆記以繁體中文撰寫。

## 範圍（嚴格遵守）

- 只收 **Java 語言與 Spring 生態** 的主題。
- 微服務、快取（Redis）、訊息佇列、Docker 等後端架構主題**不收**，
  未來由其他 `knowledge-*` repo 承接。被要求寫這類主題時，先提醒範圍邊界。

## 雙軌制

- 🔰 **基礎**（訓練主軌）：教學式筆記。每章以寫完 🔰 軌為「完整」。
- 🔬 **深入**（選修軌）：檔名以 `deep-` 開頭，寧缺勿濫，只寫最有價值的主題。
- 🔰 筆記結尾以「🔬 想深入：[標題](deep-xxx.md)」連到對應深入文（有才放）。

## 筆記結構

每篇筆記必須遵守 [TEMPLATE.md](TEMPLATE.md)，🔰 與 🔬 共用五段骨架：

- `## 前言`（工作中會遇到的提問切入）→ `## 技術背景`（先破除誤解再進主體，
  主體用 ### 展開）→ `## 實際案例` → `## 技術優缺點`（優勢 vs 代價）→
  `## 小結`（條列 3～5 條＋懸念鉤子連到 Roadmap 筆記）→
  附錄：`## 常見面試題`（1～3 題，只給方向提示）、`## 延伸閱讀`（官方文件、JEP 優先）。
- 兩軌差在「實際案例」的深度：🔰 是可執行範例與實務情境；
  🔬 是原始碼追讀／實驗／benchmark。
- 段落內規：先程式碼後解釋、對比用表格、反模式用 ❌/✅ 註解。
- 圖片選配：放該章 `attachment/img/`、以內容命名（kebab-case）；
  概念圖優先 ASCII，AI 生圖當封面並檢查圖內亂字。

## 可執行範例（混合制）

- 預設文內 snippet，力求「複製進 main 就能跑」。
- 只有「不跑看不出結果」的主題（並發競態、效能對比、GC 行為）才附範例檔：
  放該章 `examples/` 子目錄、**單一 .java 檔**、可用 `java Xxx.java` 直接執行，
  不引入 Maven/Gradle。

## README 是索引與進度表

- 新增筆記時**必須**同步更新 README 目錄表（深度、狀態兩欄）。
- 狀態：✅ 完成（符合模板）/ 🔧 待翻新（內容完成、尚未符合模板）/
  🚧 進行中 / 📝 待補。
- 翻新一篇 🔧 筆記使其符合模板後，把狀態改回 ✅。
- 章節資料夾（`NN-kebab-name/`）在該章第一篇筆記落地時才建立，不預建空資料夾。
- 寫作優先順序依 README 的 Roadmap（近期 → 中期 → 長期）。
